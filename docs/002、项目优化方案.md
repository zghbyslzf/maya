# Maya CLI 项目优化方案

## 概述

本文档基于对 Maya CLI 项目的代码分析，提出了系统性的优化方案。这些方案旨在提升代码质量、性能、可维护性和用户体验。所有优化任务已拆解为可执行的具体步骤，并按优先级和实施难度进行了分类。

---

## 🚀 快速修复（高优先级）

这些任务简单直接，风险低，可立即实施。

### 任务 1.1：修复 Clippy 警告 ✅ 已完成
- **描述**：修复 `cargo clippy` 报告的所有警告。
- **文件**：
  - `maya\crates\clear_lock\src\lib.rs`
  - `maya\crates\clear_node_modules\src\lib.rs`
  - `maya\crates\gitignore_add_zip\src\lib.rs`
  - `maya\crates\compress_pictures\src\lib.rs`
  - `maya\src\modules\clean_ops.rs`
  - `maya\src\modules\git_ops.rs`
- **具体修改**：
  - 将 `map_or(false, |name| ...)` 替换为 `is_some_and(|name| ...)`
  - 将函数参数 `&PathBuf` 改为 `&Path`
  - 移除冗余的 `use maya_common;` 导入
  - 将 `if let Ok(entry) = entry { ... }` 循环改为使用 `flatten()`

### 任务 1.2：遵循 Rust 命名约定 ✅ 已完成
- **描述**：为 `compress_pictures::ImageType::from_str` 方法实现 `std::str::FromStr` trait 或重命名。
- **文件**：`maya\crates\compress_pictures\src\lib.rs`
- **方案**：
  - 实现 `impl std::str::FromStr for ImageType`
  - 或将该方法重命名为 `from_str_safe` 等

## 🏗️ 架构改进（中优先级）

这些任务涉及代码结构的优化，提高可扩展性和可维护性。

### 任务 2.1：重构 CLI 为 Clap 子命令
- **描述**：将现有的 `if-else` 链重构为 Clap 的子命令结构。
- **文件**：`maya\src\main.rs`
- **步骤**：
  1. 定义 `enum Command`，包含 `Clean`、`Git`、`Pack`、`Optimize`、`Transform` 等变体
  2. 将现有 `struct Cli` 的字段移动到相应的子命令中
  3. 修改 `main` 函数，使用 `match` 分发子命令
  4. 更新模块调用以匹配新的参数传递方式

### 任务 2.2：统一错误处理
- **描述**：创建项目级的错误类型，替换混用的 `io::Result`、`anyhow::Result` 和直接 `eprintln!`。
- **文件**：
  - 新建：`maya\crates\maya_common\src\error.rs`
  - 修改：所有 crate 的 `lib.rs` 和 `main.rs`
- **步骤**：
  1. 在 `maya_common` 中定义 `#[derive(thiserror::Error)] enum Error`
  2. 定义 `pub type Result<T> = std::result::Result<T, Error>`
  3. 逐步替换各模块中的错误返回类型
  4. 使用 `?` 操作符进行错误传播

### 任务 2.3：提取共享的文件遍历逻辑
- **描述**：将分散在各 crate 中的文件遍历代码抽象到 `maya_common`。
- **文件**：
  - 新建：`maya\crates\maya_common\src\file_utils.rs`
  - 修改：`clear_node_modules`、`clear_lock`、`compress_pictures`、`mp4_to_m3u8` 等
- **步骤**：
  1. 实现通用函数 `find_files(dir, filter) -> Result<Vec<PathBuf>>`
  2. 实现并行版本 `find_files_parallel(dir, filter)`
  3. 在各 crate 中调用共享函数，删除重复代码

## ⚡ 性能优化（中优先级）

### 任务 3.1：并行处理图片压缩
- **描述**：使用 `rayon` 并行压缩多个图片文件。
- **文件**：`maya\crates\compress_pictures\src\lib.rs`
- **步骤**：
  1. 添加 `rayon` 依赖到 `Cargo.toml`
  2. 实现 `compress_images_parallel` 函数
  3. 在主函数中根据文件数量决定使用串行还是并行版本

### 任务 3.2：用迭代器遍历替代递归
- **描述**：将递归文件遍历改为基于 `walkdir` 的迭代器遍历，避免栈溢出风险。
- **文件**：
  - `maya\crates\clear_node_modules\src\lib.rs`
  - `maya\crates\clear_lock\src\lib.rs`
- **步骤**：
  1. 使用 `WalkDir::new(dir).into_iter()` 替换递归函数
  2. 使用 `filter_map` 处理条目
  3. 简化删除逻辑

### 任务 3.3：大文件流式处理（可选）
- **描述**：对超大图片文件使用流式处理，避免一次性加载到内存。
- **文件**：`maya\crates\compress_pictures\src\lib.rs`
- **步骤**：
  1. 为 `compress_png` 和 `compress_jpg` 实现基于 `BufReader`/`BufWriter` 的版本
  2. 根据文件大小阈值选择处理方式

## 📦 依赖管理优化

### 任务 8.1：统一依赖版本管理
- **描述**：优化工作空间的依赖声明，确保版本一致性和最小化重复。
- **文件**：`maya\Cargo.toml`
- **步骤**：
  1. 检查并统一共享依赖的版本号
  2. 确保所有 crate 都正确使用 `workspace = true` 声明
  3. 将可选特性标记为可选，减少编译时间

### 任务 8.2：移除未使用的依赖
- **描述**：清理各 crate 中未实际使用的依赖项。
- **文件**：各 crate 的 `Cargo.toml`
- **步骤**：
  1. 使用 `cargo udeps` 工具检测未使用的依赖
  2. 移除确认未使用的依赖项
  3. 更新 `Cargo.lock` 文件

### 任务 8.3：添加开发依赖管理
- **描述**：为测试和开发工具添加统一的开发依赖。

### 示例 4：统一错误处理
```rust
// 在 maya_common/src/error.rs 中
#[derive(thiserror::Error, Debug)]
pub enum Error {
    #[error("I/O错误: {0}")]
    Io(#[from] std::io::Error),
    #[error("路径错误: {0}")]
    Path(String),
    #[error("压缩错误: {0}")]
    Compression(String),
}

pub type Result<T> = std::result::Result<T, Error>;
```
